# 内存池（MemoryPool）实现与测试

## 项目简介
本项目实现了一个线程安全的固定大小内存池，用于高效管理内存分配与释放，避免频繁调用系统`malloc`/`free`带来的性能开销。内存池预先分配指定数量、指定大小的内存块，分配/释放操作均通过内存池内部的空闲链表完成，并通过互斥锁保证多线程环境下的线程安全。

## 核心功能
- 预分配固定数量、固定大小的内存块，支持快速分配/释放
- 线程安全：所有分配/释放操作均加锁保护，支持多线程并发访问
- 边界场景处理：支持极小内存块（1字节）、单块内存池等边界测试
- 内存数据完整性保证：分配的内存块可正常读写，释放后可重新分配并覆盖数据

## 核心类：MemoryPool
### 构造函数
```cpp
MemoryPool(size_t blockSize, size_t blockNum);
```
- **参数**：
  - `blockSize`：单个内存块的有效数据大小（字节）
  - `blockNum`：内存池总块数
- **功能**：预先分配 `blockNum * (blockSize + sizeof(Block))` 大小的内存，构建空闲链表

### 析构函数
```cpp
~MemoryPool();
```
- **功能**：释放内存池预分配的全部内存，加锁保证线程安全

### 核心方法
| 方法名       | 功能说明                                                                 |
|--------------|--------------------------------------------------------------------------|
| `void* alloc()` | 从内存池分配一个内存块，返回指向有效数据区域的指针；无空闲块时返回`nullptr` |
| `void free(void* ptr)` | 释放已分配的内存块，将其重新加入空闲链表；`ptr`为`nullptr`时无操作        |

## 实现细节
1. **内存块结构**：
   每个内存块包含两部分：
   - 链表节点头（`Block`结构体）：用于维护空闲链表
   - 有效数据区域：用户实际使用的内存空间
   实际单个块大小 = `sizeof(Block) + blockSize`（用户指定的块大小）

2. **空闲链表管理**：
   - 初始化时，所有内存块加入空闲链表
   - 分配时，从空闲链表头部取出块，返回有效数据区域指针
   - 释放时，将块重新插入空闲链表头部

3. **线程安全**：
   使用`std::lock_guard`+`std::mutex`实现自动加锁/解锁，保证`alloc`/`free`操作的原子性。

## 测试用例
`main.cpp` 包含4组测试用例，覆盖核心场景：

### 1. 基本分配释放测试（testBasicAllocFree）
- 验证内存池最大容量分配、无空闲块时分配失败
- 验证释放后内存块可重新分配

### 2. 并发分配释放测试（testConcurrentAllocFree）
- 多线程（默认4线程）并发分配/释放内存
- 验证线程安全，无死锁、内存泄漏

### 3. 边界场景测试（testEdgeCases）
- 块数量为1的场景：分配→分配失败→释放→重新分配成功
- 极小块大小（1字节）的分配/分配失败验证

### 4. 内存数据完整性测试（testDataIntegrity）
- 分配内存块并写入64位测试数据，验证读取数据与写入一致
- 释放后重新分配，验证数据可正常覆盖

## 注意事项
1. 内存池仅支持固定大小的内存块分配，不支持动态调整块大小/数量
2. 释放的指针必须是从当前内存池分配的有效指针，否则会导致未定义行为
3. 内存池析构时会释放所有预分配内存，无需手动释放单个块
4. 多线程场景下，锁机制会带来少量性能开销，但远低于频繁调用系统`malloc`/`free`
